IDOR Vulnerability Information

Insecure Direct Object References (IDOR) occur when an application provides direct access to objects based on user-supplied input without proper authorization checks.

Vulnerable Endpoints:

  • /api/user/{id} - Access or modify any user's data by changing the ID
  • /api/post/{id} - Access, modify, or delete any post by changing the ID

Once you're logged in, try accessing another user's data by manipulating the ID parameters in API requests!
